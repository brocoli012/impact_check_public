## 8. 사용자 플로우

### 플로우 1: 최초 설정 (1회)

```
1. PM이 Claude Code Desktop 또는 Antigravity에서 Kurly Impact Checker 스킬 설치
2. `/impact config` 실행 → LLM 프로바이더 선택 및 API 키 입력
3. API 키 유효성 검증 → 성공 시 "설정 완료" 안내
4. `/impact init {프로젝트_경로}` 실행 → 대상 프로젝트 Git 레포 경로 지정
5. 자동 인덱싱 시작 → 진행률 표시 (예: "파일 스캔 중... 342/1,205")
6. 인덱싱 완료 → 프로젝트 구조 요약 표시
   예: "✓ 화면 23개, API 엔드포인트 87개, 컴포넌트 156개 인덱싱 완료"
```

### 플로우 2: 영향도 분석 (주요 반복 사용 시나리오)

```
1. PM이 `/impact analyze` 실행
2. "기획 내용을 입력해주세요" 안내 → PM이 기획 내용을 텍스트로 입력
   예: "상품 상세 페이지에 리뷰 필터 기능을 추가하려고 합니다.
        사용자가 별점별, 최신순, 사진 리뷰만 필터링할 수 있어야 합니다."
3. (선택) PDF 파일 첨부 가능
4. 기획서 파싱 결과 표시:
   "다음과 같이 이해했습니다:
    - 대상 화면: 상품 상세 페이지
    - 주요 기능: 리뷰 필터 (별점별, 정렬, 사진 리뷰)
    맞습니까? [Y/n]"
5. PM 확인 → 영향도 분석 시작
6. 분석 진행 상황 표시:
   "코드 인덱스 매칭 중... → 영향 화면 식별 중... → 난이도 분석 중..."
7. 분석 완료 → 결과 요약 표시:
   "■ 영향도 분석 완료
    - 종합 점수: 32점 (Medium)
    - 영향 화면: 3개
    - 총 작업 수: 8개 (FE 5개, BE 3개)
    - 기획 확인 필요: 2건
    - 상세 결과를 확인하시겠습니까? [웹 보기 / 텍스트 보기 / 티켓 생성]"
8. PM 선택에 따라:
   - "웹 보기" → 시각화 웹 자동 실행 (localhost)
   - "텍스트 보기" → 터미널에 상세 결과 출력
   - "티켓 생성" → MD 티켓 파일 생성
```

### 플로우 3: 시각화 웹 확인

```
1. PM이 `/impact view` 실행 또는 분석 완료 후 "웹 보기" 선택
2. 로컬 웹 서버 자동 실행 → 브라우저 자동 오픈 (http://localhost:포트)
3. 종합 대시보드 표시:
   - 상단: 종합 점수 32점, Medium 등급 (노란색 뱃지)
   - 영향 화면 3개, 작업 8개, FE/BE 비율 차트
4. 플로우차트 뷰에서 계층 구조 확인:
   - 시스템 → 상품 상세, 리뷰 목록, 마이페이지 화면 노드
   - 각 화면 노드 클릭 → 하위 기능/모듈 전개
5. 특정 모듈 노드 클릭 → 상세 패널 표시:
   - 작업 내용, 변경 파일 경로, 난이도 점수 산출 근거
6. 필터 적용 (예: BE만 보기) → 해당 노드만 하이라이트
7. (선택) 리포트 내보내기 → MD 파일로 결과 저장
```

### 플로우 4: 인덱스 업데이트

```
1. 개발팀이 코드 변경 후 PM이 git pull 실행
2. `/impact analyze` 실행 시 자동으로 변경 감지
   "인덱스 업데이트가 필요합니다. 변경 파일 12개 감지. 업데이트 중..."
3. 부분 인덱싱 완료 → 최신 상태 반영
4. 이전과 동일하게 영향도 분석 진행
```

### 플로우 5: 에러 발생 시 복구 (Error Recovery) [피드백 R6 반영]

> PM이 기술적 에러를 직접 해결하기 어려우므로, 각 에러 상황별 명확한 안내와 복구 경로를 제공해야 합니다.

```
■ 에러 유형 1: LLM API 호출 실패 (네트워크/키/과금 문제)
1. 분석 진행 중 LLM API 호출 실패 감지
2. 에러 원인 분류 (네트워크 / API 키 만료 / 과금 한도 초과)
3. PM 친화 메시지 표시:
   "분석 서버와 연결에 실패했습니다.
    원인: API 키가 만료된 것으로 보입니다.
    해결: /impact config 에서 새 API 키를 등록하세요.
    이전까지의 분석 결과는 자동 저장되어 있습니다."
4. 키 갱신 후 `/impact analyze --resume`으로 중단 지점부터 재개 가능

■ 에러 유형 2: 인덱싱 실패 (지원하지 않는 프로젝트 구조)
1. 인덱싱 과정에서 파서가 특정 파일 처리 실패
2. 실패한 파일을 건너뛰고 나머지 인덱싱 계속 진행 (Graceful Degradation)
3. 완료 후 경고 표시:
   "인덱싱 완료 (일부 파일 제외)
    정상 처리: 1,180개 / 실패: 25개
    실패 파일은 분석 결과에서 제외됩니다.
    상세 로그: .impact/logs/indexing-errors.log"

■ 에러 유형 3: PDF 파싱 실패
1. PDF 내용을 읽지 못하는 경우 (암호화/이미지 PDF 등)
2. 안내 메시지:
   "PDF 내용을 읽을 수 없습니다.
    가능한 원인: 암호화된 PDF 또는 이미지로만 구성된 PDF
    대안: 텍스트를 직접 입력하거나, 텍스트 복사가 가능한 PDF를 사용하세요."
3. 텍스트 직접 입력 모드로 자동 전환 제안

■ 에러 유형 4: 분석 중간 타임아웃 (3분 초과)
1. 분석 시간이 예상보다 초과될 경우 중간 보고:
   "분석이 예상보다 오래 걸리고 있습니다. (현재 4분 경과)
    원인: 영향 화면이 예상보다 많거나, LLM 응답이 느린 상황입니다.
    [계속 대기] [현재까지 결과만 보기] [취소]"
2. "현재까지 결과만 보기" 선택 시 부분 결과를 표시 (미완료 항목 표시)
```
