## 1. 프로덕트 비전

### 한 줄 비전
> "기획서 한 장이면, 개발 영향도가 한눈에 보인다."

### 해결하려는 문제
컬리 PM이 새로운 기능을 기획할 때, 해당 기능이 기존 시스템의 어디에 어떤 영향을 미치는지 파악하는 데 많은 시간과 수작업이 필요합니다.

- **현재**: 기획서 작성 후, 개발팀에 구두/문서로 영향도 확인 요청 -> 개발자가 코드를 직접 살펴보며 파악 -> 수일 소요 -> 누락 빈번
- **목표**: 기획서를 입력하면 프로젝트 소스코드를 자동 분석하여 영향 범위를 화면/기능 단위로 파악하고, 난이도를 점수화하며, 시각적 플로우차트로 결과를 제공

### 목표 사용자
- **1차**: 컬리 PM팀 7명
- **역할**: 신규 기능 기획 시 개발 영향도를 빠르게 파악하고, 일정/우선순위/리소스를 합리적으로 결정해야 하는 사람
- **환경**: 로컬 PC (macOS/Windows), Claude Code Desktop 또는 Antigravity 사용

## 3. 핵심 요약 (Quick Reference)

### 배포 형태
- Claude Code Desktop / Antigravity **스킬** (독립 앱 아님)
- 슬래시 커맨드: `/impact init`, `/impact analyze`, `/impact view`, `/impact tickets` 등

### 기술 스택
- **코어 엔진**: TypeScript (Node.js)
- **시각화 웹**: React SPA + Express.js (localhost)
- **LLM**: Anthropic Claude, OpenAI GPT, Google Gemini (3개)
- **데이터**: 로컬 파일 (`.impact/` 디렉토리)

### MVP 핵심 기능 (20개)
1. 스킬 설치 및 호출 시스템 (CRITICAL)
2. 프로젝트 등록 및 코드 인덱싱 (CRITICAL)
3. Git 변경 감지 기반 자동 업데이트 (HIGH)
4. 기획서 입력 - 텍스트 (CRITICAL) + PDF (HIGH)
5. 영향도 분석 엔진 - 4 Layer 다층 분석 (CRITICAL)
6. 난이도 점수화 + 등급 산출 (CRITICAL)
7. 작업 티켓 생성 MD (HIGH)
8. 시각화 웹 - 플로우차트 (CRITICAL) + 대시보드 (HIGH)
9. LLM 멀티 프로바이더 3개 (HIGH)
10. 보강 주석 자동 생성 + 증분 업데이트 (HIGH)
11. 정책 사전 + 담당자 매핑 (MEDIUM)

### 점수 체계
- **4차원**: 개발 복잡도(35%) + 영향 범위(30%) + 정책 변경(20%) + 의존성 위험도(15%)
- **등급**: Low(1~15) / Medium(16~40) / High(41~70) / Critical(71+)
- **신뢰도**: High(85%+) / Medium(65~84%) / Low(40~64%) / Very Low(0~39%)

### 시각화 계층
- 요구사항 > 시스템 > 화면 > 하위기능 > 모듈 (5단계)
- 8종 노드: 요구사항, 시스템, 화면, 기능, 모듈, 기획확인, 정책변경, 정책경고

### 성공 기준
- PM 7명 중 5명이 주 1회 이상 사용
- 기획서 입력 후 3분 이내 결과 제공
- 영향 화면 식별 정확도 80% 이상
