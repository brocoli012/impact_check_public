## 2. CLI 인터페이스 디자인

### 2.1 명령어 구조 및 응답 형식

CLI 인터페이스는 터미널/채팅 기반으로 동작합니다. 모든 출력은 고정폭 텍스트로 표현되며, 유니코드 박스 드로잉 문자와 이모지를 활용하여 시각적 구분을 제공합니다.

#### `/impact init` - 프로젝트 등록 및 최초 인덱싱

**입력 형식**:
```
/impact init /path/to/project
```

**입력 형식**:
```

```
또는
```

```

```
  텍스트를 직접 입력하거나, PDF 파일을 첨부할 수 있습니다.

  입력 방법:
  - 텍스트: 기획 의도를 자연어로 입력
  - PDF:   /impact analyze --file 파일경로.pdf
```

> [피드백 R6 반영] **분석 대기 중 UX 강화**: PM이 분석 완료까지 3분을 기다리는 동안의 경험을 개선합니다. (1) 각 단계에 PM이 이해할 수 있는 한 줄 설명을 추가하여 "현재 무엇을 하고 있는지" 표시, (2) 화면 단위 진행 시 "N개 중 M번째 화면 분석 중" 형태로 세부 진행률 제공, (3) "경과 시간 / 예상 남은 시간" 형태로 잔여 시간을 명시, (4) 예상 비용을 진행 중에도 실시간 업데이트하여 PM이 중간에 중단 여부를 판단 가능하도록 합니다.

```
> [피드백 R6 반영] **결과 요약 카드에 "다음 단계" 맥락 안내 추가**: 분석 완료 후 PM이 단순히 W/T/K 중 선택만 하는 것이 아니라, "기획 확인 사항이 N건 있으므로 개발 요청 전에 먼저 확인하세요"와 같은 맥락 있는 안내를 제공합니다. 등급에 따라 차별화된 안내를 표시합니다:
> - Low: "일반 스프린트에 포함하여 진행하시면 됩니다."
> - Medium: "기획 확인 사항을 먼저 정리 후 개발팀에 전달하세요."
> - High: "별도 미팅을 통해 개발팀과 범위/일정을 협의하세요."
> - Critical: "경영진 보고 및 아키텍처 리뷰를 먼저 진행하세요."
```

등급별 시각 표현 (터미널 색상 지원 시):

| 등급 | 표시 | ASCII 대체 |
|------|------|-----------|
| Low | 초록색 텍스트 + `[Low]` | `[ Low ]` |
| Medium | 노란색 텍스트 + `[Medium]` | `[Medium]` |
| High | 주황색 텍스트 + `[High]` | `[ High ]` |
| Critical | 빨간색 텍스트 + `[CRITICAL]` | `[CRITICAL]` |

### 2.4 에러/확인 메시지 형식

**에러 메시지**:
```
  오류: API 키가 유효하지 않습니다.

  확인 사항:
  1. API 키가 올바르게 입력되었는지 확인하세요.
  2. API 키의 사용 기한이 만료되지 않았는지 확인하세요.

  해결 방법: /impact config --provider anthropic --key [새 API키]
```

**경고 메시지**:
```
  주의: 인덱스가 7일 전에 생성되었습니다.

  최신 코드로 인덱스를 업데이트하면 더 정확한 분석이 가능합니다.
  업데이트: /impact reindex
```

**확인 메시지**:
```
  성공: 영향도 분석이 완료되었습니다.

  결과 파일: .impact/results/2026-02-13_review-filter.json
  시각화 웹: /impact view
```

**사용자 확인 요청**:
```
  확인이 필요합니다:

  이전 분석 결과가 존재합니다. 새로 분석하면 기존 결과를 덮어쓰게 됩니다.
  계속하시겠습니까? [Y/n]
```
