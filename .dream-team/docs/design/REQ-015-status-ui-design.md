## 1. 설계 개요

### 1.1 디자인 원칙 (상태 UI 고유)

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **상태 최소 표현** | active는 기본 상태이므로 시각적 노이즈를 주지 않는다. 비정상 상태만 눈에 띄게 표시 |
| 2 | **되돌림 불가 명시** | archived 전환은 단방향이므로, 파괴적 액션에 준하는 경고를 제공한다 |
| 3 | **기존 배지 패턴 계승** | `[예시]`, `[보완]` 등 기존 inline 배지 패턴과 동일한 시각 문법을 사용한다 |
| 4 | **필터 투명성** | 상태 필터로 인해 제외된 항목이 있으면 그 사실을 반드시 사용자에게 알린다 |

### 1.2 상태별 색상 체계 (Tailwind 기반)

모든 색상은 Tailwind CSS v4 팔레트를 사용한다. 프로젝트에서 이미 사용 중인 색상 톤과의 충돌을 피하기 위해 다음과 같이 할당한다.

| 상태 | 의미 색상 | 배경 | 텍스트 | 보더 | 근거 |
|------|----------|------|--------|------|------|
| `active` | 없음 (기본) | - | - | - | 기본 상태이므로 별도 표시 불필요 |
| `completed` | 회색 (중립) | `bg-gray-100` | `text-gray-500` | `border-gray-300` | 완료 = 더 이상 활성이 아님을 톤다운으로 표현 |
| `on-hold` | 노란색 (주의) | `bg-amber-50` | `text-amber-600` | `border-amber-300` | 보류 = 주의가 필요하지만 위험하지 않음 |
| `archived` | 회색 + 반투명 | `bg-gray-50` | `text-gray-400` | `border-gray-200` | 폐기 = 비활성을 가장 강하게 표현 |

> **R8 검토**: amber는 Gap Health Widget의 MEDIUM 카드(`bg-amber-50`)와 동일 톤. 의미적으로도 "주의"로 일관된다. gray는 LOW 카드(`bg-gray-50`)와 일관.

## 2. LNB ResultCard 상태 배지

### 2.1 상태별 ResultCard 표현

#### active (진행중) - 기본

별도 배지 없음. 현재 ResultCard 그대로 표시된다. 기존과 완전히 동일한 모습이므로 사용자 학습 비용이 없다.

#### completed (완료)

카드 전체를 회색 톤다운 처리하고, 등급 옆에 체크 아이콘을 표시한다.

#### on-hold (보류)

기획서명 앞에 노란색 일시정지 배지를 표시한다.

#### archived (폐기)

카드 전체를 반투명 처리하고, 기획서명에 취소선을 적용한다.

### 2.2 와이어프레임 - ResultCard 상태별 비교

```
[active] 기본 상태 - 배지 없음:
+------------------------------+
| C   2026-02-20               |
|                              |
| 장바구니 리뉴얼               |
|                              |
| 총점 62/100                  |
| 영향 화면 3개 . 작업 12건     |
+------------------------------+
 ^-- border-l-4 transparent

[completed] 완료 - 회색 톤다운 + 체크:
+------------------------------+
| C (v) 2026-02-20            |
|                              |
| [완료] 장바구니 리뉴얼        |
|                              |
| 총점 62/100                  |
| 영향 화면 3개 . 작업 12건     |
+------------------------------+
 ^-- border-l-4 border-gray-300
     opacity-70 전체 톤다운

[on-hold] 보류 - 노란색 배지:
+------------------------------+
| C   2026-02-20               |
|                              |
| [보류] 배송 추적 개선          |
|                              |
| 총점 45/100                  |
| 영향 화면 2개 . 작업 8건      |
+------------------------------+
 ^-- border-l-4 border-amber-400

[archived] 폐기 - 취소선 + 반투명:
+------------------------------+
| C   2026-02-20               |
|                              |
| [폐기] ~결제 로직 개선~       |
|                              |
| 총점 35/100                  |
| 영향 화면 1개 . 작업 4건      |
+------------------------------+
 ^-- border-l-4 border-gray-200
     opacity-50 반투명
```

### 2.3 배지 디자인 토큰

상태 배지는 기존 `[예시]`, `[보완]` 배지와 동일한 레이아웃 패턴을 따른다.

```
기존 패턴:
[예시] = inline-flex items-center px-1.5 py-0.5 mr-1
         text-xs font-medium
         bg-purple-100 text-purple-700 rounded

[보완] = inline-flex items-center px-1.5 py-0.5 mr-1
         text-xs font-medium
         bg-violet-100 text-violet-700 rounded

신규 상태 배지:
[완료] = inline-flex items-center px-1.5 py-0.5 mr-1
         text-xs font-medium
         bg-gray-100 text-gray-600 rounded
         + 앞에 체크 아이콘 (w-3 h-3)

[보류] = inline-flex items-center px-1.5 py-0.5 mr-1
         text-xs font-medium
         bg-amber-50 text-amber-700 rounded
         + 앞에 일시정지 아이콘 (w-3 h-3)

[폐기] = inline-flex items-center px-1.5 py-0.5 mr-1
         text-xs font-medium
         bg-gray-100 text-gray-400 rounded
         + 앞에 아카이브 아이콘 (w-3 h-3)
```

> **R6 검토**: `[예시]`(`bg-purple-100`), `[보완]`(`bg-violet-100`), `[완료]`(`bg-gray-100`), `[보류]`(`bg-amber-50`), `[폐기]`(`bg-gray-100`) -- 모두 동일한 `inline-flex px-1.5 py-0.5 text-xs font-medium rounded` 구조. 배지 패턴 완전 일관.

### 2.4 ResultCard 상태별 스타일 상세

| 상태 | `border-l-4` 색상 | 카드 opacity | 등급 텍스트 | 제목 스타일 | 배지 |
|------|-------------------|:-----------:|-----------|-----------|------|
| `active` | `transparent` (기본) | 1.0 | 원래 색상 | 기본 | 없음 |
| `completed` | `border-gray-300` | 0.7 | `text-gray-400` | 기본 | `[완료]` |
| `on-hold` | `border-amber-400` | 0.85 | 원래 색상 | 기본 | `[보류]` |
| `archived` | `border-gray-200` | 0.5 | `text-gray-300` | `line-through text-gray-400` | `[폐기]` |

### 2.5 LNB 상태 필터

LNB 상단의 정렬 드롭다운 옆에 상태 필터 드롭다운을 추가한다.

```
기존:
+------------------------------------------+
| [v 최신순 ^]                              |
+------------------------------------------+

개선:
+------------------------------------------+
| [v 최신순 ^]          [v 진행중만 ^]       |
+------------------------------------------+

필터 옵션:
  - 진행중만 (active) <- 기본값
  - 전체 (active + completed + on-hold + archived)
  - 진행중 + 보류 (active + on-hold)
```

**와이어프레임 - LNB 필터 영역:**

```
+---------------------------------------------+
| [<]                                          |
|---------------------------------------------|
| [검색 (제목, 등급, 날짜)              (x)]  |
|---------------------------------------------|
| [v 최신순  ^]        [v 진행중만 ^]          |
|---------------------------------------------|
|                                              |
| (ResultCard 목록)                            |
|                                              |
+---------------------------------------------+
```

필터 드롭다운 스타일: 기존 정렬 드롭다운과 동일.
```
flex-1 text-xs px-2 py-1.5
bg-white dark:bg-gray-800
border border-gray-300 dark:border-gray-600
rounded focus:ring-2 focus:ring-blue-500
```

## 3. 상태 변경 UI

### 3.1 상태 변경 위치 결정

상태 변경은 **두 곳**에서 할 수 있다:

| 위치 | 방식 | 용도 |
|------|------|------|
| **기획 분석 대시보드 (`/analysis`)** | 상단 드롭다운 | 결과 상세를 보면서 개별 변경 |
| **프로젝트 보드 AnalysisHistoryTable** | 행 우클릭 컨텍스트 메뉴 or 인라인 드롭다운 | 목록에서 빠르게 일괄 변경 |

> **R7 검토**: 가장 빈번한 시나리오는 "분석 결과를 확인한 후 상태를 변경"이므로, 기획 분석 대시보드 상단이 1차 변경 위치다. 프로젝트 보드의 이력 테이블은 여러 건을 훑으면서 변경하는 2차 위치.

### 3.2 기획 분석 대시보드 - 상태 변경 드롭다운

ScoreHeader 영역 우측에 상태 변경 드롭다운을 배치한다. 현재 상태를 배지로 보여주고, 클릭하면 가능한 전환 옵션이 나타난다.

**와이어프레임 - ScoreHeader 내 상태 드롭다운:**

```
+-----------------------------------------------------------------------+
| [ProjectSelector: v eSCM-Front ^]                                      |
|-----------------------------------------------------------------------|
|                                                                       |
|  +----+                                                               |
|  | C  |  "장바구니 리뉴얼"                  상태: [v 진행중  ^]         |
|  +----+  총점 62/100                                                  |
|                                                                       |
+-----------------------------------------------------------------------+
                                                |
                                                v  (클릭 시 드롭다운)
                                         +--------------------+
                                         | (v) 진행중 (active) |
                                         |---------------------|
                                         |  완료 처리           |
                                         |  보류 처리           |
                                         |---------------------|
                                         |  폐기 (되돌릴 수 없음)|
                                         +---------------------+
```

### 3.3 드롭다운 옵션 및 전환 제한

상태 전환 규칙(PRD 2.2절)에 따라, 현재 상태에서 전환 가능한 옵션만 표시한다.

| 현재 상태 | 사용 가능 옵션 | 비활성 옵션 |
|----------|--------------|------------|
| `active` | 완료 처리, 보류 처리, 폐기 | - |
| `completed` | 폐기 | 진행중 (불가: 보완 분석으로 대체), 보류 (불가) |
| `on-hold` | 재개 (-> active), 폐기 | 완료 (불가: active 경유 필수) |
| `archived` | (없음 - 드롭다운 비활성) | 전부 불가 |

**archived 상태일 때:**

```
상태: [폐기됨]    (드롭다운 아닌 정적 배지)

[폐기됨] = inline-flex items-center px-2 py-1
           text-xs font-medium
           bg-gray-100 text-gray-400 rounded
           cursor-not-allowed
```

드롭다운 대신 정적 배지만 표시하여, 상태 변경이 불가능함을 시각적으로 전달한다.

### 3.4 상태 변경 확인 다이얼로그

일반 전환(active -> completed, active -> on-hold 등)에는 확인 없이 즉시 반영한다. **단, 폐기(archived) 전환만 확인 다이얼로그를 표시한다.**

**와이어프레임 - 폐기 확인 다이얼로그:**

```
+-----------------------------------------------+
|  분석 결과 폐기                          [x]    |
|-----------------------------------------------|
|                                               |
|  "장바구니 리뉴얼" 분석 결과를                 |
|  폐기하시겠습니까?                             |
|                                               |
|  [!] 폐기된 분석은 되돌릴 수 없습니다.          |
|      보완 분석 제안 및 gap 탐지에서              |
|      영구적으로 제외됩니다.                      |
|                                               |
|              [취소]     [폐기 처리]             |
|                                               |
+-----------------------------------------------+
```

스타일:
- 오버레이: `fixed inset-0 bg-black/50 z-50`
- 다이얼로그: `bg-white rounded-lg shadow-xl p-6 max-w-sm mx-auto mt-[20vh]`
- 경고 영역: `bg-red-50 border border-red-200 rounded p-3 text-sm text-red-700`
- 취소 버튼: `px-4 py-2 text-sm text-gray-700 border border-gray-300 rounded hover:bg-gray-50`
- 폐기 버튼: `px-4 py-2 text-sm text-white bg-red-600 rounded hover:bg-red-700`

> **R7 검토**: 일반 전환은 1클릭 즉시 반영 (최소 동선). 폐기만 2클릭 (1. 드롭다운에서 "폐기" 선택 + 2. 확인 다이얼로그에서 "폐기 처리" 클릭). 되돌릴 수 없는 액션에 대한 안전장치로 적절.

### 3.5 상태 변경 성공 토스트

상태 변경 API 호출 성공 시 토스트 알림을 표시한다.

```
+----------------------------------------------+
|  (v) 상태가 변경되었습니다.                     |
|      "장바구니 리뉴얼": 진행중 -> 완료          |
+----------------------------------------------+
```

토스트 스타일: 기존 CLI 명령어 복사 토스트와 동일.
```
bg-gray-900 text-white rounded-lg shadow-lg p-3
text-sm
자동 사라짐: 3초
```

### 3.6 프로젝트 보드 AnalysisHistoryTable - 인라인 상태 변경

이력 테이블의 상태 컬럼에서 상태 배지를 클릭하면 인라인 드롭다운이 열린다. 이 방식은 테이블에서 여러 건을 빠르게 변경할 때 유용하다.

**와이어프레임 - 테이블 인라인 상태 변경:**

```
+------------------------------------------------------------------------+
| #  기획서명                    등급  점수  상태         분석일            |
|----|--------------------------|-----|------|------------|----------------|
| 1  장바구니 리뉴얼             [C]    62   [v 진행중 ^]  2026-02-20     |
| 2  결제 로직 개선              [B]    45   [완료]        2026-02-18     |
| 3  배송 추적 개선              [B]    40   [보류]        2026-02-15     |
| 4  회원 등급 개편              [A]    28   [폐기됨]      2026-02-10     |
+------------------------------------------------------------------------+
```

- `[v 진행중 ^]`: active 상태 -> 클릭 시 드롭다운 표시 (3.3절과 동일한 전환 규칙)
- `[완료]`: completed 배지로 표시. 클릭 시 "폐기" 옵션만 있는 드롭다운
- `[보류]`: on-hold 배지로 표시. 클릭 시 "재개", "폐기" 옵션
- `[폐기됨]`: archived 정적 배지. 클릭 불가 (`cursor-not-allowed`)

## 4. 분석 이력 테이블 상태 컬럼

### 4.1 상태 컬럼 추가

기존 AnalysisHistoryTable의 헤더에 "상태" 컬럼을 추가한다. 분석일 앞에 배치하여, 등급/점수와 상태를 함께 파악할 수 있게 한다.

**기존 헤더:**
```
| #  기획서명    등급  점수  분석일 |
```

**개선 헤더:**
```
| #  기획서명    등급  점수  상태      분석일 |
```

### 4.2 상태 컬럼 셀 디자인

| 상태 | 셀 내용 | 스타일 |
|------|--------|--------|
| `active` | `진행중` (텍스트만) | `text-xs text-gray-500` |
| `completed` | `[v] 완료` (체크 아이콘 + 텍스트) | `text-xs text-gray-500 bg-gray-100 rounded px-1.5 py-0.5` |
| `on-hold` | `[||] 보류` (일시정지 아이콘 + 텍스트) | `text-xs text-amber-600 bg-amber-50 rounded px-1.5 py-0.5` |
| `archived` | `[x] 폐기됨` (아카이브 아이콘 + 텍스트) | `text-xs text-gray-400 bg-gray-50 rounded px-1.5 py-0.5` |

active는 배지 없이 텍스트만 표시하여 "기본 상태 = 시각적 무게 최소화" 원칙을 따른다.

### 4.3 와이어프레임 - 상태 컬럼 포함 전체 테이블

```
+------------------------------------------------------------------------+
| 최근 분석 이력                                          [전체 보기 ->]   |
|------------------------------------------------------------------------|
| #  기획서명                    등급  점수  상태         분석일            |
|----|--------------------------|-----|------|------------|----------------|
| 1  장바구니 리뉴얼             [C]    62   진행중        2026-02-20     |
| 2  [보완] 상품 물리 스펙->bat  [B]    45   진행중        2026-02-22     |
| 3  결제 로직 개선              [B]    45   [v] 완료      2026-02-18     |
| 4  배송 추적 개선              [B]    40   [||] 보류     2026-02-15     |
| 5  회원 등급 개편              [A]    28   [x] 폐기됨    2026-02-10     |
+------------------------------------------------------------------------+
```

### 4.4 필터링 기능

이력 테이블 상단 "최근 분석 이력" 헤더 옆에 필터 칩(chip)을 배치한다.

**와이어프레임 - 필터 칩:**

```
+------------------------------------------------------------------------+
| 최근 분석 이력  [전체] [진행중*] [완료] [보류] [폐기]   [전체 보기 ->]   |
|------------------------------------------------------------------------|
| (테이블 본문)                                                           |
+------------------------------------------------------------------------+

* = 현재 선택된 필터 (강조 표시)
```

필터 칩 스타일:

## 5. 보완 분석 제안 시 상태 필터 표시

### 5.1 제외 안내 문구 위치

```
|-----------------------------------------------------------------------|
|  (KPI 카드, 상세 목록 ...)                                             |
|                                                                       |
|  +----------------------------------------------------------------+  |
|  | (i) active 상태 분석 기준 |                                     |  |
|  |     제외: 완료 2건, 보류 1건, 폐기 0건                           |  |
|  +----------------------------------------------------------------+  |
|                                                                       |
+-----------------------------------------------------------------------+
```

안내 문구 스타일:
```
bg-gray-50 border border-gray-200 rounded p-2.5
text-xs text-gray-500
(i) 아이콘: text-gray-400 w-3.5 h-3.5
```

## 6. 기존 데이터 (상태 없는 건) 시각적 처리

### 6.1 처리 원칙

`status` 필드가 없는 기존 데이터는 `active`로 간주한다 (PRD 10.1절 Lazy Migration 전략). UI에서는 이를 active와 완전히 동일하게 표시하며, **특별한 시각적 표시를 추가하지 않는다.**

### 6.2 근거

| 선택지 | 장점 | 단점 | 결정 |
|--------|------|------|:----:|
| A. 마이그레이션 안내 배너 표시 | 사용자가 상태 체계 도입을 인지 | 기존 사용자에게 혼란, 불필요한 액션 유도 | X |
| B. `[상태 미설정]` 배지 표시 | 명시적 | 모든 기존 데이터에 불필요한 노이즈 추가 | X |
| **C. active와 동일하게 표시** | 무소음 마이그레이션, 기존 경험 유지 | 사용자가 상태 체계 도입을 모를 수 있음 | **O** |

선택지 C를 채택한다. 상태 체계는 사용자가 처음으로 상태를 변경할 때 자연스럽게 인지하게 된다. 프로젝트 보드의 이력 테이블에 상태 컬럼이 추가되면서 "진행중"이라는 기본 상태가 자연스럽게 노출된다.

### 6.3 프론트엔드 Fallback 코드 패턴

```typescript
// API 응답에서 status가 없으면 'active'로 처리
function getEffectiveStatus(result: ResultSummary): AnalysisStatus {
  return result.status ?? 'active';
}
```

이 함수를 LNB, AnalysisHistoryTable, Dashboard 등 모든 상태 참조 지점에서 사용한다.

### 7.1 상태 변경 인터랙션 흐름

| 동작 | 결과 |
|------|------|
| Dashboard 상태 드롭다운 클릭 | 가능한 전환 옵션 표시 |

| 영향 UI | 갱신 방법 |

### 7.3 로딩 상태

| 동작 | 표현 |
|------|------|

## 10. 피드백 검토 (R6~R8)

### R6: 상태 표시와 기존 배지 패턴 일관성

| 기존 배지 | 상태 배지 | 일관성 |
|----------|----------|:------:|
| `[예시]` = `bg-purple-100 text-purple-700 rounded px-1.5 py-0.5 text-xs font-medium` | `[완료]` = `bg-gray-100 text-gray-600 rounded px-1.5 py-0.5 text-xs font-medium` | O |
| `[보완]` = `bg-violet-100 text-violet-700 rounded px-1.5 py-0.5 text-xs font-medium` | `[보류]` = `bg-amber-50 text-amber-700 rounded px-1.5 py-0.5 text-xs font-medium` | O |

| ResultCard `border-l-4 border-violet-400` (보완) | `border-l-4 border-gray-200` (폐기) | O |

**결론**: 모든 상태 배지가 기존 `inline-flex px-1.5 py-0.5 text-xs font-medium rounded` 패턴을 정확히 따른다. 배지 앞 아이콘(체크, 일시정지, 아카이브)만 추가되어 시각적 구분을 강화한다.

### R7: 상태 변경 동선 최소 클릭

| 시나리오 | 클릭 수 | 경로 |
|---------|:-------:|------|

### R8: 색상 체계와 Tailwind 팔레트 일관성

| 사용 색상 | Tailwind 토큰 | 기존 사용처 | 의미 일관성 |

## 부록 A: 상태 배지 + 보완 라벨 조합 규칙

보완 분석(`isSupplement: true`)과 상태 배지가 동시에 표시될 때의 순서:

```
배지 표시 순서: [상태 배지] [보완] 기획서명

예:
  [완료] [보완] 상품 물리 스펙 수정 -> e-scm-batch
  [보류] [보완] 결제 로직 개선 -> e-scm-front

active 보완:
  [보완] 상품 물리 스펙 수정 -> e-scm-batch
  (active는 배지 없으므로 보완만 표시)

active 일반:
  장바구니 리뉴얼
  (배지 없음)
```

규칙:
1. 상태 배지가 보완 배지보다 앞에 온다 (상태가 더 상위 개념)
2. active 상태는 배지를 표시하지 않으므로, 보완 배지만 노출
3. 예시(`isDemo`) + 상태 조합은 발생하지 않음 (예시 데이터에는 상태 개념 없음)
