# EPIC-001: Kurly Impact Checker

## 개요
기획서/텍스트 기반으로 개발 작업 영향도를 분석하고, 작업 티켓을 생성하며, 로컬 웹에서 플로우차트 형태로 시각화하는 PM 지원 도구

## 배경
- 컬리 PM이 새로운 기능 요구사항을 받으면 기획서 기반으로 작업 영향도를 체크하여 일정/우선순위를 결정해야 함
- 현재는 수동으로 영향도 파악 → 비효율적
- 코드 기반 자동 분석 + LLM 활용으로 효율화 필요

## 핵심 기능
1. 프로젝트 등록 & 코드 인덱싱 (Git 레포 분석, 변경 감지 자동 업데이트)
2. 기획서 입력 & 분석 (PDF/Confluence/텍스트/이미지)
3. 작업 티켓 생성 (MD → 추후 Jira 연동)
4. 영향도 시각화 웹 (시스템>화면>하위기능>모듈, 난이도 점수화, 플로우차트)
5. LLM 멀티 프로바이더 (3~4개 지원, 사용자별 API 키)

## 사용자
- 컬리 PM팀 (초기 7명)

## 기술 조건
- 실행 환경: 로컬 PC
- 기획서 형식: PDF, Confluence, 텍스트 입력 (이미지 첨부 가능)
- 기획서 평균 10페이지 이내
- MVP에서는 외부 도구 연동 없음 (Jira/GitHub은 추후)
- LLM: 비용/성능 밸런스, 3~4개 프로바이더 지원, 사용자별 API 키 인증
