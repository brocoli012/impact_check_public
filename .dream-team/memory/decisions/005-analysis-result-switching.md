## [2026-02-15] 분석 결과 전환 기능 - MVP 이후 진행 결정

### 논의 참여자
- Master, 사용자

### 논의 내용
- 사용자: 기획서나 영향도 요청이 여러 개일 경우 선택하면서 전환하는 기능이 필요
- Master 조사 결과: 백엔드 인프라(API, ResultManager, Zustand 상태, Hook)는 모두 구현 완료. 프론트엔드 UI(선택 드롭다운/셀렉터)만 없는 상태
- 기존 기획: MVP Scope 문서에서 "분석 히스토리 비교"를 Phase 2(MVP 이후)로 분류

### 최종 결정
- MVP 완성 후 별도 작업으로 진행
- **단순 드롭다운이 아닌, 디자이너에게 사용성 및 디자인 검토를 거쳐 진행**
- 디자이너가 UX 관점에서 최적의 전환 방식을 제안하도록 함 (드롭다운, 사이드 패널, 히스토리 타임라인 등)

### 사용자 관점 검토
- PM이 여러 기획서를 순차적으로 분석할 때 결과를 비교/전환하는 것은 핵심 사용성
- 단순 드롭다운은 사용성이 부족하므로 디자이너 리뷰 필수
- 백엔드는 이미 준비되어 있어 프론트엔드 UI 작업만 필요

### 진행 시기
- MVP (Phase 5) 완료 이후
- 드림팀 프로세스: Master → PM → Designer → TPO → Developer → QA
